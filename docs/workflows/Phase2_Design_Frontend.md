# FASE 2: DESIGN SYSTEM & FRONTEND

## 🎯 Objetivo de la Fase
Definir y diseñar la identidad visual, el Design System, y los prototipos de alta fidelidad para FUTX STORE, asegurando una experiencia de usuario orientada a conversiones.

## 📦 Entregables Específicos
1. Design System Completo (Tipografía, Colores, Espacios, Componentes Reutilizables).
2. Wireframes de páginas clave (Homepage, Categorías, Producto, Checkout, Cart).
3. Mockups High-Fidelity para Homepage, PDP (Product Detail Page), y Checkout.

## 🤖 Prompts para Ejecutar (Copiar y Pegar)

**Prompt 2.1: Design System Definition**
```text
@frontend-design Eres diseñador senior de UI/UX. Crea el design system base para FUTX STORE:
Colores: Verde #1a472a (principal), Dorado #D4AF37 (premium), Rojo #E63946 (urgencia), Blanco/Gris.
Tipografía: Montserrat + Inter/Roboto.
Genera especificaciones (tokens CSS) para:
1. Paleta de colores con roles.
2. Escala tipográfica.
3. Sistema de espaciado.
4. Definición de componentes core: Header, Footer, Botones (Primary, Secondary, Urgency), Product Cards.
5. Reglas de accesibilidad básicas (WCAG).
```

**Prompt 2.2: Wireframes y Flujos**
```text
@frontend-design Elabora el layout (en formato markdown o pseudocódigo) para los wireframes móviles y de escritorio de:
1. Homepage (Hero, Beneficios, Colecciones, Social Proof, Bestsellers, Urgencia).
2. Product Detail Page (Galería, Precio/Stock, CTA, Detalles, Reviews).
3. Cart y Flujo de Checkout en 3 pasos.
Enfatiza las áreas de conversión y mitigación de fricciones (envío gratis, garantía 30 días).
```

## ✅ Checklist de Completitud
- [ ] Design tokens documentados.
- [ ] Wireframes de las 8 páginas core creados.
- [ ] Revisión de contraste y accesibilidad.
- [ ] Mobile-first approach verificado en mockups.

## ⚠️ Riesgos y Mitigación
- **Riesgo:** Experiencia pobre en móvil.
- **Mitigación:** Diseñar y prototipar 100% "Mobile First".
- **Riesgo:** Checkout con demasiada fricción que causa abandono.
- **Mitigación:** Checkout en 3 pasos claros, mantener el carrito siempre visible y con métodos de pago exprés.

## 📅 Timeline Realista
- **Duración:** 5 Días (Semana 1 - Semana 2)
- Días 3-4: Design System y Wireframes.
- Días 5-7: Mockups de alta fidelidad y ajustes finales de UI.
